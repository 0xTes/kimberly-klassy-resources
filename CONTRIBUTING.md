# Contributing

Thanks for improving this template. Keep changes small, focused, and easy to review.

## Local workflow

1. Fork or create a branch from the default branch.
2. Run `npm install`, then `npm run dev`.
3. Run `npm run lint` and `npm run build` before opening a pull request.
4. Explain the user-facing effect of the change and include screenshots for visual changes.

## Code guidelines

- Keep creator-specific content in `src/config/brand.config.js`.
- Prefer small components with one clear responsibility.
- Use semantic HTML, visible focus styles, and keyboard-friendly interactions.
- Do not add dependencies when simple React, CSS, or HTML can solve the problem.
